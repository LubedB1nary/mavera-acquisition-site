# Document 5: Website Design Brief for Claude Design

> **This document is the brief.** It tells Claude Design everything it needs to know about Mavera, the purpose of the new website, the audiences, the messaging hierarchy, the content structure, and the desired outcomes. It does NOT prescribe visual direction, layout, or specific design decisions — that's Claude Design's job.

---

## SECTION A: WHO YOU'RE DESIGNING FOR

### A.1 The Company

**Mavera** is an AI-powered intelligence platform that helps companies grow. It combines synthetic audience intelligence (AI personas that behave like real people, validated at 98% accuracy against Harvard's OASIS benchmark), autonomous AI agents (that execute full business workflows independently), and an enterprise-grade API (OpenAI SDK-compatible, used by 9 enterprise customers including pipeline with Coinbase, S&P Global, PitchBook, Spotify, Hasbro, and H&R Block).

Mavera holds SOC 2 Type II, ISO 27001, and ISO 42001 certifications. The platform processes research through 295K+ media sources and 50M+ annotated behavioral interactions.

The company has two co-founders:
- **Alex Hassan** — CTO, sole technical builder, built the entire platform
- **Jill Axline** — CEO, PhD (Cornell), former Morningstar marketing executive

### A.2 The Products (Three, Unified Under One Brand)

1. **The Mavera Platform** (app.mavera.io) — A web-based SaaS where teams run synthetic audience research, test content, analyze video, monitor news, generate content, deploy autonomous sales agents, and conduct AI-powered research. Credit-based pricing ($300-$3,500/mo).

2. **The Mavera API** (docs.mavera.io) — A fully documented, OpenAI SDK-compatible API that exposes every platform capability programmatically. 11 feature areas, SDKs for Python/JS/Go, 200+ workflow templates. Sold to developers AND non-developers (the thesis: tools like Claude Code, Cursor, and Codex have made APIs accessible to everyone).

3. **Maven** — A native macOS desktop app that acts as an AI coworker. It integrates the Mavera API alongside iMessage, Apple Mail, contacts, and other tools. Maven can text you, schedule autonomous tasks, track long-lived goals, and run AI agents while you sleep. Maven is the premium, local-first experience layer.

### A.3 The Audiences (Who Will Visit This Website)

**Primary: Growth-Oriented Business Leaders**
- Founders, CMOs, VPs of Sales, Heads of Growth
- They don't care about prompts or APIs — they care about outcomes: more customers, faster research, better decisions, measurable ROI
- They want to see: "What will this do for my business?" not "What features does it have?"
- They're evaluating Mavera against: hiring more people, using traditional research firms, or doing nothing

**Secondary: Technical Evaluators & Developers**
- CTOs, engineering leads, developers building integrations
- They care about: API compatibility (OpenAI SDK drop-in), documentation quality, SDKs, rate limits, pricing transparency
- They want to see: code examples, architecture, and a path to docs.mavera.io
- They're evaluating Mavera against: building it themselves, using OpenAI directly, or using competitors

**Tertiary: Enterprise Procurement & Compliance**
- Security teams, legal, procurement
- They care about: SOC 2, ISO certifications, data privacy, HIPAA readiness, audit trails
- They want to see: trust portal, compliance documentation, evidence of enterprise customers
- They're evaluating Mavera against: risk

**Emerging: Non-Developer API Users**
- Marketers, researchers, analysts who use Claude Code, Cursor, or Codex to interact with APIs
- They don't write code traditionally but can use AI coding tools to call APIs
- They want to see: that the API is simple, that there are templates, that they don't need to be engineers
- This is a growing and underserved audience

## SECTION B: THE PURPOSE OF THIS WEBSITE

### B.1 What This Website Must Accomplish

1. **Communicate the shift from tool to platform.** Mavera is not "another AI marketing tool." It's an autonomous growth platform. The website must make visitors feel that Mavera is in a different category than the tools they've seen before.

2. **Lead with outcomes, not features.** The hero should not say "AI-powered marketing intelligence." It should communicate a growth outcome. The user's first thought should be "this could change my business" not "this has a lot of features."

3. **Showcase autonomous agents as the headline capability.** The 7 AI sales agents (Signal Scanner, Prospect Researcher, Outreach Drafter, Conversation Handler, Qualifier, Re-engager, Meeting Prepper) are the most differentiated feature. They should be prominently featured — not buried in a features page.

4. **Make the API feel accessible to non-developers.** The API is a major revenue driver, and the audience is expanding beyond traditional developers. The website should communicate that the API is OpenAI-compatible (familiar), well-documented, and usable by anyone with access to modern AI coding tools.

5. **Build trust for enterprise buyers.** SOC 2, ISO 27001, ISO 42001, 9 enterprise customers, pipeline with household names. This credibility needs to be visible without being the lead story.

6. **Drive three actions:** Start a free trial / Book a demo / Explore the API docs.

### B.2 What This Website Must NOT Do

- Do NOT lead with features or technical specifications
- Do NOT use jargon like "LLM," "multi-model swarm," "MCP," or "synthetic data" in hero copy
- Do NOT position Mavera as a developer tool — it's a business tool with a developer-friendly API
- Do NOT make it feel like a startup landing page — Mavera has enterprise customers and certifications; it should feel established and premium
- Do NOT use the word "prompt" in customer-facing copy — the framing is outcomes, not prompts

## SECTION C: MESSAGING HIERARCHY

### C.1 The Core Message

**Mavera opens the door to growth.**

The platform doesn't ask you to write better prompts. It asks you what growth outcome you want — more customers, better retention, higher conversion, faster research — and then deploys autonomous agents backed by synthetic intelligence to get there. Every recommendation comes with evidence you can defend.

### C.2 Messaging Layers (in order of importance)

**Layer 1 — The Outcome Promise**
- "Open the door to growth"
- Growth is the outcome. Mavera is the mechanism. Evidence is the proof.
- The visitor should immediately understand: this platform helps my business grow, and it can prove it.

**Layer 2 — Autonomous Agents**
- "AI agents that work while you sleep"
- Not copilots. Not chatbots. Autonomous agents that detect signals, research prospects, draft outreach, handle conversations, qualify leads, and prep meetings — independently.
- The visitor should understand: I don't have to do the work. The agents do.

**Layer 3 — Synthetic Intelligence**
- "Audiences you can ask before you spend"
- 50+ AI personas validated at 98% accuracy against real human panels. Test any message, any creative, any strategy against your actual audience — instantly, at a fraction of the cost.
- The visitor should understand: I can know what my audience thinks before I commit budget.

**Layer 4 — Evidence & Traceability**
- "Every decision comes with receipts"
- Every recommendation has sources. Every agent action is logged. Every score is explainable. Every output is auditable.
- The visitor should understand: I can defend every decision Mavera helps me make.

**Layer 5 — Platform Breadth**
- Research, content, video, news, meetings, sales, API
- The visitor should understand: this isn't a point solution — it's a platform that covers the full growth cycle.

**Layer 6 — Enterprise Trust**
- SOC 2 Type II, ISO 27001, ISO 42001, 9 enterprise customers, Coinbase/S&P/Spotify pipeline
- The visitor should understand: this is safe, proven, and used by companies I recognize.

### C.3 Tone & Voice Direction

- **Confident but not arrogant.** Mavera has real customers, real certifications, and real results. State facts, don't hype.
- **Outcome-oriented, not feature-oriented.** "Grow faster" not "Our AI uses multi-model architecture."
- **Calm authority.** Think Stripe, Linear, or Vercel — companies that let the product speak. Not aggressive SaaS landing pages with countdown timers.
- **Growth language.** "Open doors," "unlock," "accelerate," "scale," "compound." Not "disrupt," "revolutionize," "game-changing."

## SECTION D: CONTENT STRUCTURE (WHAT THE SITE NEEDS TO COMMUNICATE)

### D.1 Hero Section

**Primary message:** An outcome-oriented headline about growth. Something in the spirit of "Open the door to growth" — the idea that Mavera removes the barriers between a company and its growth potential.

**Supporting message:** Brief explanation of how — autonomous agents + synthetic intelligence + evidence-backed decisions.

**Social proof:** Enterprise customer logos or "Trusted by 9 enterprise customers" with recognizable pipeline names.

**CTAs:** Primary (Start free / Get started) + Secondary (Book a demo or Watch it work)

### D.2 The Autonomous Agents Story

This should be the first major section after the hero. Show the 7 agents and the autonomous sales cycle:

**The narrative arc:**
1. Signals are detected in your market (Signal Scanner)
2. Prospects are researched automatically (Prospect Researcher)
3. Personalized outreach is drafted and sent (Outreach Drafter)
4. Replies are handled intelligently (Conversation Handler)
5. Leads are qualified against your criteria (Qualifier)
6. Stalled deals are re-engaged with fresh angles (Re-engager)
7. You show up to meetings fully prepared (Meeting Prepper)

**The key insight to communicate:** This isn't automation (doing the same thing faster). This is autonomy (doing the thinking, the research, the writing, the qualifying — independently). The human closes the deal. Everything before that is handled.

### D.3 Synthetic Intelligence / Persona Research

**The narrative:** Before you spend budget, ask your audience. Mavera's 50+ AI personas are statistically validated against real human panels (98% accuracy, OASIS benchmark). Run focus groups in minutes instead of weeks. Test content before it goes live. Know what resonates before you commit.

**Key capabilities to reference:**
- 50+ pre-built personas across generational, professional, lifestyle, industry, and expert categories
- Custom persona creation
- Focus groups with 12 question types
- Content testing with structured engagement scores
- Video analysis with emotional/cognitive/behavioral metrics
- 96% parity with traditional research panels (52+ published studies)

### D.4 The Evidence Layer

**The narrative:** Every decision comes with receipts. Mavera doesn't give you opinions — it gives you evidence. Sources are cited. Confidence is scored. Reasoning is transparent. Outputs are auditable. In a world of AI hallucination, Mavera's outputs are defensible.

**Key capabilities to reference:**
- Mave Agent's 5-phase research with source attribution and hallucination risk scoring
- Analysis mode with confidence scores, emotional metrics, and cognitive bias detection
- Full audit trails on every agent action
- Explainable lead scoring (every score shows contributing factors)

### D.5 The Platform Overview

**The narrative:** One platform for the full growth cycle — from market intelligence to audience research to content creation to sales execution to meeting intelligence.

**Feature areas to reference (briefly — this is overview, not deep dive):**
- Responses API (persona-enhanced AI)
- Mave Agent (deep research)
- Personas (50+ built-in + custom)
- Focus Groups (synthetic audience research)
- Video Analysis (creative testing)
- News Intelligence (market monitoring)
- Brand Voice (consistent tone)
- Content Generation (template-based)
- Meetings (recording + AI analysis)
- Autonomous Sales Agents (the Glengarry module)
- CRM (full pipeline management)

### D.6 The API / Developer Section

**The narrative:** OpenAI SDK-compatible. Change your base URL, add a persona, and your existing code works with Mavera. SDKs for Python, JavaScript, and Go. 200+ workflow templates. Full documentation at docs.mavera.io.

**Key points:**
- Drop-in replacement for OpenAI's Responses API
- `mvra_live_` API keys
- Model: `mavera-1`
- Credit-based pricing (transparent, every response shows cost)
- Not just for developers — anyone using Claude Code, Cursor, or Codex can use the API

**Code example to show:**
```python
from openai import OpenAI

client = OpenAI(
    api_key="mvra_live_...",
    base_url="https://app.mavera.io/api/v1",
)

response = client.responses.create(
    model="mavera-1",
    input="How do millennials feel about remote work?",
    extra_body={"persona_id": "gen_z_consumer"},
)
```

### D.7 Trust & Enterprise

**The narrative:** Built for enterprise from day one.

**Proof points:**
- SOC 2 Type II

## SECTION F: WHAT SUCCESS LOOKS LIKE

The new website succeeds if:

1. **A founder visits and thinks:** "This could be my growth engine. I need to try this."
2. **A CTO visits and thinks:** "This is a real platform with a real API. I can integrate this."
3. **A VP of Sales visits and thinks:** "These autonomous agents could replace 3 SDRs."
4. **A marketer visits and thinks:** "I can test my campaigns against real audiences before I spend."
5. **An enterprise buyer visits and thinks:** "SOC 2, ISO, real customers — this is safe to buy."
6. **A non-developer visits and thinks:** "I don't need to be an engineer to use this."

The website should feel like opening a door — not to a product demo, but to a new way of growing a business. Calm, confident, evidence-backed, and unmistakably premium.

## SECTION G: IMPORTANT TERMINOLOGY

| Use This | Not This | Why |
|----------|----------|-----|
| Growth outcomes | Prompts | Outcome-oriented, not tool-oriented |
| Autonomous agents | AI assistants / copilots | Agents work independently; assistants wait for instructions |
| Synthetic intelligence | AI personas / chatbots | "Synthetic intelligence" conveys the depth of the behavioral modeling |
| Evidence / receipts | Data / analytics | "Evidence" implies defensibility; "data" is generic |
| Platform | Tool | Mavera is a platform with multiple capabilities, not a single tool |
| Mavera | Maven (on the website) | Maven is the desktop app; the website is for the Mavera platform and API |
| Credits | Tokens | Mavera uses credits, not tokens — simpler, more business-friendly |
| Growth engine | Marketing tool | Mavera does sales, research, content, meetings — not just marketing |
